# Security Guidelines & Official Channels

Apollo Care is an open, community‑governed protocol.  To protect yourself from scams and impersonators, follow these guidelines:

## Official Domains & Contacts

* **Website:** [apollocare.io](https://apollocare.io) is the only official website.  Sub‑domains of apollocare.io may host dashboards or apps.  Beware of typosquatted domains.
* **Token Address:** The one and only official $APH token address on Solana is `6S3T6f1mmhxQWKR1gMiZR1SZLpu396jnnRMGqAZUj3Qj`.  Verify this address on a Solana explorer before interacting with any token or contract.
* **Repository:** The GitHub repository you are viewing is the canonical source of protocol documentation.  Contracts and addresses will only be published here or via verified DAO governance proposals.

## Anti‑Phishing Tips

* **Do not share private keys, seed phrases or API keys with anyone.** Apollo will never ask for your wallet seed or private key.  If someone requests this information, it is a scam.
* **Verify links** before connecting your wallet.  Only use official URLs posted by the DAO or this repository.
* **Double‑check contract addresses**.  Fraudsters may publish fake $APH contracts or pools.  Always verify that addresses match those published in this repo and on the DAO’s official channels.
* **Beware of “admin assistance” offers.** Apollo has no support agents who will “help” transfer your tokens or upgrade your wallet.  All protocol upgrades occur via public DAO proposals.
* **Use trusted explorers.** To verify token balances and contracts, use reputable explorers like Solscan or the Solana Explorer.  Cross‑check addresses and transaction hashes.

## Responsible Disclosure

If you discover a vulnerability or security issue in Apollo’s smart contracts or infrastructure, please report it responsibly via the official channels (e.g., by filing a security issue in this repo or contacting the core contributors via the email published on apollocare.io).  Do not post exploits or private information publicly.

## No Sensitive Data

Apollo does not publish private keys, seed phrases, multisig signer lists or any operational secrets.  All governance‑controlled wallets use multisig setups; their addresses are published only after contracts are deployed.  Never trust any address posted outside the DAO’s official communications.
