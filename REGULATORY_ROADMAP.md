## 2. Phase 1: Healthcare Sharing Ministry

### 2.1 Overview

| Attribute | Details |
|-----------|---------|
| **Structure** | Healthcare Cost Sharing Ministry (HCSM) |
| **Legal Basis** | ACA §5000A exemption |
| **Oversight** | Self-regulated (not state insurance) |
| **Timeline** | Launch through Year 1-2 |

### 2.2 How HCSMs Work

Healthcare Sharing Ministries are member-funded organizations where participants voluntarily share medical expenses. Key characteristics:

- **Not insurance** — Members share costs, no contractual guarantee
- **Voluntary participation** — Sharing is discretionary
- **Community-based** — Members agree to shared values
- **Exempt from ACA** — Not subject to insurance regulations

### 2.3 Required Disclosures

Phase 1 members receive clear disclosures:

```
┌─────────────────────────────────────────────────────────────────┐
│                    PHASE 1 DISCLOSURES                          │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ⚠️  This is NOT insurance.                                      │
│                                                                 │
│  ⚠️  Sharing of medical expenses is voluntary.                   │
│                                                                 │
│  ⚠️  There is no guarantee that your expenses will be shared.    │
│                                                                 │
│  ⚠️  This program is not subject to state insurance regulations. │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2.4 Apollo's Differentiation

Unlike traditional HCSMs, Apollo provides:

| Feature | Traditional HCSM | Apollo HCSM |
|---------|------------------|-------------|
| **Transparency** | Limited | Full on-chain visibility |
| **Reserve reporting** | Often none | Real-time CAR |
| **Claim decisions** | Opaque | AI + committee with appeals |
| **Governance** | Top-down | DAO democratic |
| **Technology** | Manual processes | Blockchain automation |

### 2.5 Phase 1 Parameters

| Parameter | Value |
|-----------|-------|
| **Maximum individual claim** | $100,000 |
| **Pre-existing condition exclusions** | Allowed (waiting period) |
| **Coverage terminology** | "Monthly share amount" |
| **Target membership** | 100-1,000 members |
| **Geographic scope** | Nationwide |

### 2.6 Exit Criteria

To transition to Phase 2:

| Requirement | Threshold |
|-------------|-----------|
| Active members | 1,000+ |
| Operating history | 12+ months |
| Total capital | $2M+ |
| Demonstrated MLR | ≥85% |
| CAR maintained | ≥125% |
| DAO approval | 67% supermajority |
